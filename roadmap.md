# Learning Roadmap — Claude Code to Revenue

**Goal:** Financial and geographic independence by building and selling
AI-powered services to businesses.

**Strategy:** Learn only what's needed to build and sell. Deepen skills
when real projects demand it. Ship fast, iterate on feedback.

---

## Phase 1: Build Foundation (Week 1) ✅
> Know your tools well enough to build with them.

- [x] Complete Anthropic Skilljar course
- [x] Claude Code workflow: ask, edit, review, commit
- [x] CLAUDE.md — how to shape Claude's behavior per project
- [x] Git basics: init, add, commit, push
- [x] Push learning-claude repo to GitHub
- [x] Build a CLI tool (file organizer), review the code
- [x] Make your first API call (ad copy generator with Claude API)
- [x] API key security: .env files, .gitignore

**Status:** Done. Can navigate projects, use Claude Code, push to GitHub, call APIs.

---

## Phase 2: Build Sellable Tools (Weeks 2-3)
> Build things that solve real business problems. These become your portfolio AND your products.

### Project 1 — AI Ad Copy Generator (enhanced)
- [ ] Add more platforms (TikTok, Google Display)
- [ ] Accept a brief/context file instead of just a one-liner
- [ ] Output to a formatted file (not just terminal)
- [ ] Test with real MeglerBio and agency use cases

### Project 2 — Client-Facing Web Tool
- [ ] Build a simple web app using Claude (Lovable, Next.js, or static HTML)
- [ ] Connect it to the Claude API so clients can interact with it
- [ ] Deploy it live (Vercel, Netlify, or Lovable)
- [ ] This becomes a portfolio piece AND a lead magnet

### Project 3 — Business Automation
- [ ] Build one automation that saves real time (content generation, file processing, report creation)
- [ ] Use n8n or scripts — whichever is faster for the use case
- [ ] Document it as a case study: problem → solution → result

**Milestone:** You have 2-3 working tools you can demo to potential clients.

---

## Key Principles

1. **Build to sell, not to learn.** Learning is a byproduct of building real things.
2. **Use Claude Code for everything.** You direct, it writes. Your job is reviewing and prompting.
3. **Ship fast, iterate later.** A working ugly tool beats a perfect plan.
4. **Document as you go.** Every project is a potential case study.
5. **Learn just-in-time.** Don't study topics until a real project demands it.
